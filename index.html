2. Anleitung zum Selbermachen

Anstelle dieser Anleitung kannst Du auch diesen Spickzettel (GoogleDoc) verwenden.

2.1 Basis-Variante

Um ein erstes, einfaches Twine zu erstellen, folgst Du diesen Schrtten:

  1. Öffne die Websie twinery.org
  2. Erstelle eine neue Geschichte und gib ier einen passenden Namen.
  3. Erstelle eine erste Passage, benenne sie möglichst aussagekräftig (z.B. start) und gib Deinen gewünschten Inhalt ein. Verlinkungen haben die folgende Form: [[Mein verlinkter Text ->passagenname]].
  4. Wenn Du fertig bist, dann öffne das Menü und wähle 'Als Datei veröffentlichen'. Es wird eine .html Datei heruntergeladen.
  5. Teile die HTML-Datei (z.B. Versand via Mail) oder stelle sie auf Filebin.net

Optional kannst Du das folgende Basis-Stylesheet kopieren, auf Wunsch anpassen und dann in Dein Twine einfügen (Menü -> Stylesheet der Geschichte bearbeiten

html { font-family: helvetica, arial, sans-serif; background-color: #396fc6; color: black; }
tw-story { width: 100%; font-size: 2em; line-height: 2em; background-color: white; color: #13102d; }
tw-sidebar {display:none}
tw-link, .enchantment-link { color: #396fc6; text-decoration: none; }

Ebenso kannst Du die Stylesheets aus den oben verlinkten Beispielen nutzen (Stylesheet der Twine-Übersicht und Stylesheet der Geschichte 'Im Zauberwald'). Auf diese Weise sieht Dein Twine gleich deutlich schicker aus.

2.2 Profi-Variante

Die oben dargestellte Basis-Variante ist ein guter Anfang. Aber mit Twine kannst Du noch viel mehr machen. Mit wenigen Befehlen (Texteingaben) lässt sich in einem Twine viel bewirken. 5 Best-Of Befehle stelle ich hier vor. Du findest sie zusammengestellt in diesem Twine.

  1. Twines lassen sich personalisieren, indem man eine ‘Namensabfrage’ an den Anfang stellt. Alternativ kann eine Auswahl vorgegeben und daraus gewählt werden:
    (set: $name to (prompt: ‘Bitte gib Deinen Namen ein:’, ‘...’))
    (cycling-link: bind $name, 'Max', 'Maria', 'Markus', 'Mascha', 'Mara')
  2. Twines lassen sich bei jedem Lesen zufallsmäßig ändern und/ oder der Text wechselt in einem festgelegten Abstand.
    Der Keller war (either: ‘dunkel’, ‘stickig’, ‘verstaubt’, ‘hell’, ‘kalt’, ‘finster’)
    Der Keller war (live: 1s)[(either: ‘dunkel’, ‘stickig’, ‘verstaubt’, ‘hell’, ‘kalt’, ‘finster’)]
  3. Text kann in Twine vielfältig dargestellt werden. Beispiele
    (text-style: "shadow")[Hallo. Wie geht es Dir?]
    (text-style: "mark")[Hallo. Wie geht es Dir?]
    (text-style: "outline")[Hallo. Wie geht es Dir?]
    (text-style: "blur")[Hallo. Wie geht es Dir?]
    (text-style: "blink")[Hallo. Wie geht es Dir?]
    Und auch vielfältig erscheinen: Beispiel (t8n: "pulse")[Hallo, wie geht es Dir?] (Ebenso geht z.B. flicker, dissolve, rumble, shudder oder slide-right
  4. Auch die Übergänge zur nächsten Folie können vielfältig gestaltet werden, z.B. 'zitternd'. Beispiel:
    (t8n-depart: "rumble")[[slide4]]
  5. Inhalte können nach bestimmten Regeln angezeigt werden:
    (event: when time > 3s)[Text erscheintnach 3 Sekunden]
    (link:"Klick")[Dann erscheint weiterer Text]
    (more:)[Text erscheint, wenn keine anderen Links mehr da sind.]

3. Veröffentlichung

Ein Twine kann lokal gespeichert und dann im Browser geöffnet werden. Wenn man es im Netz veröffentlichen und via Link teilen will, geht das für eine Woche via Filebin.net. Dauerhaft kann ein Twine auf Glitch oder Github kostenfrei gespeichert werden. Wer Wordpress nutzt, kann ein Twine über den Media-Upload hochladen.
